claude‑youtube‑editor user guide

Terminal-driven video editing

You record the talking head. Claude Code does the rest.

This isn't an app you click through. You drop a recording into the project and talk to Claude Code in your terminal — it cuts the footage, builds animated on-screen visuals as code, cleans your voice, places sound effects, and packages the result. This guide is the shortest path from an empty project to a rendered video.

00 See it work right now — free, no footage

Before recording anything, watch the visual engine run. From the repo root:

# from D:\Projects\Claude Projects\video-edit
cd remotion
npm run studio

Open http://localhost:3000 in Chrome or Edge. You'll see 38 example shots from a real published video — click any to play it. Every one renders from code with no camera and no API key. This is what "visuals built as code" means, and it's the fastest way to understand what the tool produces.

01 How it works

Three moving parts, and you only ever touch the first one directly:

You + Claude Code

You type plain requests in the terminal (clean cut videos/video-1). Claude picks the matching skill and runs the tools for you. There is no dashboard — the conversation is the interface.

Python tools

The real work — transcription, cutting, denoising, sound design, thumbnails — happens in tools/*.py, using the project's venv and ffmpeg.

Remotion (visuals)

On-screen animations — title cards, code, fake screencasts, diagrams — are React/TSX "shots" in remotion/, rendered and composited over your cut.

One rule

Run every command from the repo root, and use the venv Python (venv/Scripts/python on Windows) — not a bare python.

Do this once, first

Run /brand-setup (or say "set up my brand"). It asks for your colors, fonts, and wordmark, then rewrites the three style files together so your videos look like your channel instead of the demo's. Skip it and everything comes out in the house style.

1

Clean cut — get this right before anything else

clean cut videos/video-1

AssemblyAI transcribes your footage; Claude reads the transcript and decides what's a real point versus a false start or filler, writing a clean master plus edited-transcript.json — the word-by-word timing map every later step depends on. A bad cut fails downstream silently, so review it in the cut editor before moving on.

2

Visual beats

add TSX beats to video-1

Builds animated overlays and full-screen segments synced to your word times. The /fake-screencast technique turns static screenshots into a simulated screen recording — animated cursor, changing URL bar, scrolling — so you never open a screen recorder.

3

Clean the voice

clean audio for video-1

Diagnoses the noise and isolates your voice while preserving your levels. This is the one step with a free local option (RNNoise) if you'd rather not pay for ElevenLabs.

4

Sound effects

suggest sfx for video-1

Plans each cue on an exact word into sfx-plan.json. There's a hard stop: it won't mix until you audit the cue sheet.

5

Packaging

package this video

Produces one locked title plus three thumbnail bets for YouTube's built-in A/B test. Face thumbnails need your own reference photos in media/library/faces/.

6

Verify, then upload

venv/Scripts/python tools/verify_cut.py videos/video-1   # half-cut words + A/V drift
venv/Scripts/python tools/yt_upload.py auth              # one-time browser login
venv/Scripts/python tools/yt_upload.py upload <plan.json>

It uploads as a private draft, so you review before anything goes live.

03 Two ways to begin

Edit existing footage

You already have a talking-head recording, voiceover, screen recording, or demo. Drop the file into videos/video-1/ and run the pipeline above starting at step 1. This is the tool's core, working path today.

Faceless from a source

Turn a public GitHub repo, an article URL, or pasted text into an animated video with no camera — assembled from the same Remotion visuals. This is the Module 1 capability now being built (see the status section); today it's driven by Claude rather than a one-click button.

05 Command cheat sheet

Talk to Claude in plain language for the pipeline; the raw commands below are for the demo, setup, and manual steps. Always from the repo root.

# see the visual engine (no footage, no keys)
cd remotion && npm run studio        # → http://localhost:3000

# after adding or renaming a Remotion shot, regenerate the registry
cd remotion && npm run gen

# render the example shots to video files
cd remotion && npm run render

# type-check the Remotion project (the only quality gate that exists)
cd remotion && npx tsc --noEmit

# open the cut editor for a project (the review gate in step 1)
venv/Scripts/python tools/editor/server.py videos/video-1   # → http://localhost:8765

In Claude Code, the plain-language equivalents: set up my brand · clean cut videos/video-1 · add TSX beats to video-1 · clean audio for video-1 · suggest sfx for video-1 · package this video.

06 Troubleshooting

ffmpeg: command not found

ffmpeg was just added to your PATH — restart your terminal (and Claude Code) so it's picked up. New windows will find it. The Studio demo doesn't need ffmpeg, so it works regardless.

ModuleNotFoundError (requests, Pillow, google-…)

You ran a bare python that isn't the project venv. Use venv/Scripts/python tools/<tool>.py … (Windows) or activate the venv first.

Known limitation — GPU required for the cut render

The final cut render (render_cuts.py) currently hard-requires an NVIDIA GPU (NVENC) with no CPU fallback — it fails on most laptops and all Macs. The visuals bake (bake.py) uses the CPU and works anywhere. A CPU fallback for the cut render is the first fix queued in Module 1 (see below).

Paths and projects

Pass projects as videos/video-1 and run from the repo root — tools resolve project paths against your working directory. Raw footage and master cuts are intentionally kept out of git.

07 Where things live

tools/       # Python pipeline tools + the cut-editor UI (tools/editor/)
remotion/    # the visual engine — src/lib/ kit, src/shots/<project>/
media/       # library/ (reusable sfx, music, faces) + projects/<project>/
videos/      # one folder per video — starts empty; drop footage here
brand.md     # the style contract (with remotion/src/brand.ts + fonts.ts)
.env         # your API keys (never committed)

08 Module 1 status

The project is being upgraded incrementally. The current focus is making footage editing easier and adding a first version of faceless video from a source (GitHub repo, article, or pasted text). A full audit of the existing code — architecture, reusable components, and the honest gaps — lives in MODULE_1_AUDIT.md, with step tracking in MODULE_1_PROGRESS.md.

Headline from the audit: this is a CLI + Claude-skills pipeline, not a web app, so the faceless feature is being built CLI-first and reuses the existing Remotion components rather than rebuilding them.
